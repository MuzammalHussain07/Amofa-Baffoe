AWS Infrastructure Architecture & VPC Design – Proof of Concept (PoC)

3-Day Technical Information Deliverable — Prepared by Muzammal Hussain

Project Owner: Amofa Baffoe Project

Overview

This document summarizes the 3-day deliverable requested by the client: a concise technical brief to confirm PoC scope, objectives and timetable, and to list all comprehensive requirements, resources and dependencies required to proceed with the 1-month PoC build.

Deliverable: 3-Day Technical Brief

Mandatory PoC Requirements

  • Confirmed PoC scope and exclusions (synthetic data only; no live banking integrations)
  • List of PoC Objectives and success criteria
  • Timetable for PoC activities (Day 1–3: brief; 1 month PoC build)
  • Comprehensive requirements, technical resources, and dependencies

Project Scope (3-Day Brief)

What is included in this 3-day deliverable (documentation only):

  • High-level architecture summary (VPC, subnets, API Gateway, Lambda, S3, RDS/Redshift, QuickSight)
  • Detailed list of PoC deliverables and acceptance criteria
  • Technical resource requirements and access checklist
  • Dependencies, constraints and risks that must be resolved prior to PoC build
  • A short timetable and next-step recommendations

Objectives & Success Criteria

  • Demonstrate a secure, scalable AWS proof of concept using a single VPC
  • Verify end-to-end data flow using synthetic datasets (upload → validation → storage → visualisation)
  • Deliver actionable documentation and a slide deck for stakeholder presentation
  • Provide step-by-step tutorials to enable the client to explain and demo the PoC

Timetable — 3 Days (Immediate)

Day 1: Confirm scope, collect access info, list deliverables to include in PoC.
Day 2: Prepare comprehensive requirements, technical resources checklist, and outline architecture diagrams.
Day 3: Finalize brief, prepare slide deck and deliverable package for client review.

Comprehensive List of Requirements (Sample)

Functional

  • Data upload endpoint (CSV/JSON)
  • Data validation & processing logic
  • Dashboard views: bank metrics, trend lines, risk flags
  • File-based synthetic dataset support

Non-Functional

  • Data residency in AWS Africa (Cape Town)
  • Encryption at rest and in transit (KMS, TLS)
  • Basic monitoring & audit logging (CloudWatch, CloudTrail)
  • Role-based access control (IAM)

Technical Resources Needed

  • An AWS account (Africa - Cape Town region) with permissions to create VPC, API Gateway, Lambda, S3, RDS/Redshift, IAM, QuickSight.
  • Synthetic datasets (CSV/JSON) or sample data schema to be used for testing and dashboards.
  • Project owner contact and (optional) identity provider details for SSO/IAM mapping.
  • Access window and point-of-contact for any firewall/VPC whitelist requirements.

Dependencies & Risks

  • Timely provision of AWS access and QuickSight entitlements (delays will affect timeline).
  • Availability of representative synthetic datasets (or schema) to validate data flow.
  • Client security/compliance approvals for even sandbox environments.
  • Limitations of PoC resources (size/performance) vs production expectations.

Deliverable Package (what you will receive)

  • 3-page Technical Brief (PDF)
  • 6–10 slide presentation deck (PPTX) suitable for stakeholder briefing
  • Comprehensive requirements checklist & technical resource list
  • Step-by-step tutorial notes the client can use during the PoC presentation

Next Steps (recommended)

  1. Confirm the 3-day brief scope and approve the deliverable package.
  2. Provide AWS account access and synthetic datasets as listed above.
  3. Review and approve the final brief and slide deck (Day 3) so PoC build can begin.
